BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding
Devlin et al., Google AI Language, 2018

ABSTRACT
We introduce BERT (Bidirectional Encoder Representations from Transformers). Unlike recent language representation models, BERT is designed to pre-train deep bidirectional representations from unlabeled text by jointly conditioning on both left and right context in all layers. As a result, the pre-trained BERT model can be fine-tuned with just one additional output layer to create state-of-the-art models for a wide range of tasks — including question answering and language inference — without substantial task-specific architecture modifications.

BERT advances the state of the art for eleven NLP tasks:
- GLUE: 80.5% (+7.7%)
- MultiNLI: 86.7% (+4.6%)
- SQuAD v1.1 F1: 93.2 (+1.5)
- SQuAD v2.0 F1: 83.1 (+5.1)

1. INTRODUCTION
Language model pre-training has been shown to be effective for improving many NLP tasks. These include sentence-level tasks such as natural language inference and paraphrasing, as well as token-level tasks such as named entity recognition and question answering.

There are two existing strategies for applying pre-trained language representations to downstream tasks:
(1) Feature-based: uses task-specific architectures with pre-trained representations as features (e.g., ELMo).
(2) Fine-tuning: introduces minimal task-specific parameters and is trained on the downstream tasks by fine-tuning all pre-trained parameters (e.g., OpenAI GPT).

A major limitation of current techniques is that standard language models are unidirectional, which limits the choice of architectures during pre-training. For example, in OpenAI GPT, the authors use a left-to-right architecture. This is sub-optimal for sentence-level tasks and harmful for token-level tasks like question answering, where it is crucial to incorporate context from both directions.

BERT alleviates this by using a masked language model (MLM) pre-training objective. The MLM randomly masks some tokens from the input, and the objective is to predict the original vocabulary id of the masked word based only on its context.

2. BERT MODEL
BERT's model architecture is a multi-layer bidirectional Transformer encoder. We primarily report results on two model sizes:

BERT_BASE:
  - Layers (L): 12
  - Hidden size (H): 768
  - Attention heads (A): 12
  - Total parameters: 110M

BERT_LARGE:
  - Layers (L): 24
  - Hidden size (H): 1024
  - Attention heads (A): 16
  - Total parameters: 340M

2.1 Pre-training Tasks
Task 1 — Masked Language Model (MLM):
15% of input tokens are masked at random. The model predicts these masked tokens. This bidirectional pre-training is more powerful than left-to-right pre-training.

Task 2 — Next Sentence Prediction (NSP):
The model receives pairs of sentences and learns to predict whether the second sentence follows the first. This helps with tasks like QA and NLI that require understanding of sentence relationships.

2.2 Pre-training Data
BooksCorpus (800M words) + English Wikipedia (2,500M words).
Only text passages extracted; lists, tables, and headers discarded.

2.3 Fine-tuning
For classification tasks: a [CLS] token classification head is added.
For span prediction (SQuAD): start/end token classification heads are added.
Fine-tuning is inexpensive — most results can be replicated in at most 1 hour on a single Cloud TPU.

3. KEY DIFFERENCES FROM GPT
- GPT is trained on BooksCorpus only; BERT is trained on BooksCorpus + Wikipedia.
- GPT uses a sentence separator [SEP] and classifier token [CLS] only at fine-tuning; BERT learns them during pre-training.
- GPT is trained for 1M steps with a batch size of 32,000 words; BERT trains for 1M steps with 128,000 words per batch.
- GPT uses the same learning rate for all fine-tuning experiments; BERT uses task-specific fine-tuning rates.

4. RESULTS SUMMARY
BERT_LARGE sets new state-of-the-art on every benchmark tested:
- GLUE benchmark: BERT_LARGE = 80.5%
- SQuAD 1.1: 93.2 F1 / 87.4 EM
- SQuAD 2.0: 83.1 F1 / 80.0 EM
- Named Entity Recognition (CoNLL-2003): 92.8 F1
- SWAG (grounded inference): 86.3% accuracy

5. ABLATION STUDIES
Effect of removing NSP: -3.5% on MultiNLI, -2.0% on SQuAD 1.1.
Effect of LTR vs bidirectional: BERT_BASE bidirectional >> LTR on SQuAD by +4.9 F1.
Effect of model size: larger models improve across all tasks, even for small datasets.

6. CONCLUSION
BERT is a new language representation model that obtains state-of-the-art results on a wide array of NLP tasks. The key technical novelty is the application of bidirectional pre-training to language representations via the Masked Language Model objective. The model can be fine-tuned for downstream tasks with minimal additional architecture changes and achieves competitive results with much less labeled data than task-specific models trained from scratch.

LIMITATIONS
- Large memory footprint (BERT_LARGE: 340M parameters).
- Slow inference due to model size.
- The [MASK] token is never seen during fine-tuning, creating a pre-training/fine-tuning discrepancy.
- NSP task has later been shown to provide limited benefit (RoBERTa finding).

IMPACT & CITATIONS
BERT has been cited over 70,000 times and spawned numerous variants: RoBERTa, ALBERT, DistilBERT, SpanBERT, BioBERT, SciBERT, and many more.
